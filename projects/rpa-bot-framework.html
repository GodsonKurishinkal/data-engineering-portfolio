The Challenge

Enterprise supply chains run on legacy systems. ERPs from the 2000s, warehouse management systems with Java applets, and reporting tools that only export to Excel. These systems hold critical data but offer no APIs.

Our data team faced a daily grind:

  • Manually logging into 5+ systems every morning
  • Navigating complex UI workflows to generate reports
  • Downloading Excel files, then manually importing to our lakehouse
  • 4+ hours of repetitive work before actual analysis could begin
  • Human errors causing data inconsistencies downstream

The Solution

I built a modular RPA framework that treats each legacy system as a data source. Bots handle the UI interactions, extract data, and feed it directly into our Bronze layer‚Äîno human intervention required.

Implementation

The framework uses an Abstract Base Class pattern for consistency:

base_bot.py 
from abc import ABC, abstractmethod
from selenium import webdriver
from selenium.webdriver.support.ui import WebDriverWait

class BaseBot(ABC):
    """Abstract base class for all RPA bots."""
    
    def __init__(self, config: BotConfig):
        self.config = config
        self.driver = None
        self.logger = setup_logger(self.__class__.__name__)
        
    def run(self) -> ExtractResult:
        """Main execution flow with error handling."""
        try:
            self._initialize_driver()
            self._login()
            data = self._extract()
            self._save_to_bronze(data)
            return ExtractResult(success=True, records=len(data))
        except Exception as e:
            self._capture_screenshot()
            self._send_alert(str(e))
            raise
        finally:
            self._cleanup()
    
    @abstractmethod
    def _login(self) -> None:
        """Implement system-specific login flow."""
        pass
    
    @abstractmethod
    def _extract(self) -> pd.DataFrame:
        """Implement system-specific extraction logic."""
        pass
    
    def _wait_for_element(self, locator, timeout=30):
        """Wait for element with retry logic."""
        return WebDriverWait(self.driver, timeout).until(
            EC.presence_of_element_located(locator)
        )

Key Learnings

UI Changes Break Bots

System updates change element IDs. Using semantic selectors and fallback strategies reduced maintenance by 60%.

Timing is Everything

Hard-coded waits fail. Implemented smart waits with element visibility checks and exponential backoff.

Screenshots Save Hours

Automatic screenshots on failures made debugging 10x faster‚Äîseeing exactly where the bot got stuck.

Headless Has Limits

Some legacy Java applets only work with visible browsers. Learned to detect and switch modes dynamically.
