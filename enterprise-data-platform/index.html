02

Architecture decisions

Every non-obvious choice is recorded as a lightweight ADR in docs/architecture-decisions/. The pattern — Context · Decision · Consequences — makes future-me (and future-you) able to argue with past-me.

Polars over PandasADR-001

Arrow-backed, expression-based, 5–10× faster on our Silver transforms. Narrower API means fewer ways to write the wrong query.

DuckDB for analyticsADR-002

Zero-copy Parquet reads, columnar engine, single-binary deployment. The right tool for ad-hoc Gold-layer exploration without spinning up a warehouse.

Medallion over alternativesADR-003

Bronze / Silver / Gold gave us the separation we needed without the Spark dependency Delta Lake demanded. Data Vault was overkill for the scale.
